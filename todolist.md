# OnlineMsgServer 安全改进 ToDo

## P0（必须优先完成）

- [x] 启用 WSS（TLS）替代明文 WS
  - 目标：`ws://` 改为 `wss://`，部署有效证书并强制 TLS 连接
  - 验收：抓包不可读明文业务数据；客户端证书校验通过

- [x] 增加服务端身份校验机制
  - 目标：客户端校验证书链，必要时增加证书指纹 pinning
  - 验收：中间人替换首包公钥时，客户端拒绝连接

- [x] 增加客户端身份认证
  - 目标：连接后 challenge-response（客户端私钥签名挑战值），会话绑定身份
  - 验收：未认证连接不能发送 `forward` / `broadcast`

## P1（高优先级）

- [x] 增加消息完整性与防重放
  - 目标：消息体增加 `timestamp + nonce + signature`
  - 验收：重复消息被拒绝；篡改消息验签失败

- [x] 增加滥用防护（DoS/刷消息）
  - 目标：限制单连接 QPS、消息大小、连接总数；异常触发封禁/退避
  - 验收：压测下服务可用；超阈值请求被限流

- [x] 只允许已登记公钥的会话参与收发
  - 目标：所有消息处理入口做鉴权前置检查
  - 验收：未登记公钥无法被转发目标命中，也不能广播

## P2（中优先级）

- [x] 私钥安全存储与轮换
  - 目标：禁止明文 `.pkey` 落盘，改为 KMS/Secret 挂载；制定轮换策略
  - 验收：代码与镜像中无明文私钥；轮换不影响在线通信

- [x] 容器最小权限运行
  - 目标：Docker 镜像增加非 root 用户，收敛文件与网络权限
  - 验收：容器以非 root 运行，关键目录最小可写

- [x] 安全审计日志
  - 目标：记录认证失败、限流触发、异常解密、重放拦截事件
  - 验收：可按会话/时间追踪安全事件，日志不泄露敏感明文

## 建议执行顺序

1. TLS/WSS 与服务端身份校验
2. 客户端认证与消息验签
3. 限流与防重放
4. 私钥与容器安全加固
5. 审计日志与告警接入
